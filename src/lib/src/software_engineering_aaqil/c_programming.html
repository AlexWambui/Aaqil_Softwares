Introduction

C was created by Dennis MacAlistair Richie, an American computer scientist. He created the language together with a collegue, Ken Thompson at the Bell Telephone Laboratories in 1972.

C was created for a specific purpose of designing the UNIX operating system (which is used on many computers).

C is powerful and flexible thus quickly spread beyond Bell Labs. Programmers everywhere began using it to write all sorts of programs. However, different organizations began utilizing their own versions of C, and subtle differences between implementations started to cause programmers headaches. In response to this, the American National Standards Institute (ANSI) formed a committee in 1983 to establish a standard definition of C, which became known as ANSI Standard C.

C got it's name from being a predecessor of B language. B language was developed by Ken Thompson of Bell Labs.

C is an imperative (procedural) language. All the code has to be inside the main function.

C files (normally known as source files) have to be compiled using a compiler such as gcc in order to create an executable file (machine code the computer can understand and execute).

c is a general-purpose programming language.

It's mostly useful for creating compilers, interpreters, embedded systems, device drivers, micro processors, micro controllers and operating systems (Windows, Mac OS, Linux).

C is usually the best when it comes to speed or performance but not for prototyping (that is, for exploring small amounts of data, for developing an approach, and algorithms, for analyzing data and for generating graphics).

C Standardization

These include:

  1. K&R C
  2. ANSI C - (aka C89 or C90)
    • most portable
    • to turn on the ANSCI C mode, use the command: -ansi, -std=c89
  3. C99 - extends C89/C90, new data types, variable length arrays.

The ANSI C spec standard determines the minimum values of each type.

C Program Cycle

Creating the source code

Source code - is a series of statements or commands that are used to instruct the computer to perform your desired tasks.

When you save a source file, you must give it a name.

The name should describe what the program does.

When you save a C program source file, give it a file .c extension.

Using an Editor

Most compilers come with a built-in editor that can be used to enter source code; however, some don't.

Most computer systems include a program that can be used as an editor. UNIX system: ed, ex, edit, emacs or vi. Microsoft Windows: Notepad.

Most word processors use special codes to format their documents. These codes can't be read correctly by other programs.

The American Standard Code for Information Interchange (ASCII) has specified a standard text format that nearly any program, including C, can use.

Many Word processors, such as WordPerfect, AmiPro, Word, WordPad and WordStar, are capable of saving source files in ASCII form (as a text file rather than a document file).

When you want to save a word processor's file as an ASCII file, select the ASCII or text option when saving.

Compiling the Source Code

Although you understand C source code, your computer can't. A computer requires digital, or binary, instructions in what is called machine language.

Before a C program can run on a computer, it must be translated from source code to machine language. This translation, second step in program development, is performed by a program called a compiler.

The compiler takes the source code file as input and produces a disk file containing the machine language instructions that correspond to your source code statements.

The machine language instructions created by the compiler are called object code, and the disk file containing them is called an objec file.

Each compiler needs its own command to be used to create the object code. To compile, you typically use the command to run the compiler followed by the source filename.

To compile radius.c on a UNIX machine, we would use the command:

cc radius.c

After compiling, you have an object file. There should now be a file that has the same name as your source file, but with an .obj rather than .c extension.

The .obj is recognized as an object file and is used by the linker.

In UNIX, the compiler created object files with an extension of .o instead of .obj.

Using gcc compiler.

GCC is the acronym of the GNU compiler collection.

GCC supports C, C++, Java, Ada, Go etc.

By default, GCC uses the ANSI standardization of C (C89 + a number of C99 features).

sudo apt-get install gcc -y

Compiling C source code with GCC

to compile a source code file names main.c using GCC:

This created a binary named a.out which is the file we execute.

Setting output file name

By default, after compiler operation, the created executable file name will be a.out.

We can specify the executable file name after compilation as:

Linking to create an executable file

Part of the C language is a function library that contains object code (code that has already been compiled) for predefined functions.

A predefined function contains C code that has already been written and is suppled in a ready-to-use form with your compiler package.

The printf() function is a library function.

Library functions perform frequently needed tasks, such as displaying information on-screen and reading data from disk files.

If your program uses any of these functions, the object file produced when your source code was compiled must be combined with object code from the function library to create the final executable program.

Executable means the program can be run or executed, on your computer.

The process of converting object code into executable code is called linking and it performed by a linker.

Executing code

Once your program is compiled and linked to create an executable file, you can now run it.

When you make changes to the source code, you need to recompile and relink the program to get the updated executable file.

Many compilers, such as DOS compilers do compiling and linking as one step.

Errors

Errors are an unavoidable part of program development.

Compilation Errors

A compilation error occurs when the compiler finds something in the source code that it can't compile.

A misspelling, typographical error, or any of a dozen other things can cause the compiler to choke.

Although the compiler is very clever about detecting and localizing errors, you must use your knowledge of the C language to interpret the compiler's messages and determine the actual location of any errors that are reported.

The errors reported might differ depending on the compiler. In most cases, the error message should give you an idea of what or where the problem is.

At times, a single error in a C program can sometimes cause multiple error messages. In such a case, find one error, fix it and recompile. You might find that your single correction is all that's needed, and the program will compile without errors.

Link Error Messages

Linker errors are relatively rare and usually a result of misspelling the name of a C library function.

In this case, you get an error: undefined symbols, followed by the missplled name (preceded by an underscore).

Data Types

Data types in C refer to an extensive system used for declaring variables or functions of different types.

The type of a variable determines how much space it occupies in storage and how the bit pattern stored is interpreted.

Type in C can be classified as:

  1. Basic Types - are arithmetic types and are further classified into: integer and floating-point types.
  2. Enumerated types - are arithmetic types and are used to define variables that can only assign certain discrete integer values throughout the program.
  3. Type void - the type specifier void indicates that no value is available.
  4. Derived types - they include: pointer types, array types, structure types, union types and function types.

The void type

The void type specifies that no value is available.

It's used in three kinds of situations:

  1. Function returns as void - There are various functions in C which do not return any value or you can say they return void. A function with no return value has the return type as void. eg. void exit (int status);
  2. Function arguments as void - There are various functions in C which do not accept any parameter. A function with no parameter can accept a void. For example, int rand(void);
  3. Pointers to void - A pointer of type void * represents the address of an object, but not its type. For example, a memory allocation function void *malloc(size_t size); returns a pointer to void which can be casted to any data type.

Conditionals

At times we want to check data and make choices based on the state of that data.

C provides two ways of doing so: if...else statements and switch statement.

if

An if statement allows you to check for a condition to be true, and the execute the block provided in the curly brackets. Or execute a different block if the original condition turns out to be false.

Example:

                        
                            #include 
                            int main(void)
                            {
                                int number;

                                number = printf("Enter a number: ");
                                scanf("%i", &number);

                                if (number < 2)
                                {
                                    printf("%i is less than 2\n", number);
                                }
                                else if (number > 2)
                                {
                                    printf("%i is greater than 2\n", number);
                                }
                                else
                                {
                                    printf("a is equal to 2\n");
                                }

                                return (0);
                            }

Switch

If you need to do too many if / else / if blocks to perform a check, perhaps because you need to check the exact value of a variable, then switch can be very useful.

You only need to provide a variable as a condition, and a series of case entry points for each value you expect.

Example:

                        
                            #include <stdio.h>
                            int main(void)
                            {
                                char operator;
                                int answer;
                                int x, y;
                                operator = printf("Enter operator (- + / *): ");
                                scanf("%c", &operator);
                                x = printf("X: ");
                                scanf("%i", &x);
                                y = printf("Y: ");
                                scanf("%i", &y);
                                switch (operator)
                                {
                                    case '+':
                                        answer = x + y;
                                        printf("Sum = %i\n", answer);
                                        break;

                                    case '-':
                                        answer = x - y;
                                        printf("Difference = %i\n", answer);
                                        break;

                                    case '*':
                                        answer = x * y;
                                        printf("Product = %i\n", answer);
                                        break;

                                    case '/':
                                        answer = x / y;
                                        printf("Multiple = %i\n", answer);
                                        break;        

                                    default:
                                        printf("Ooops!!! Sorry. I'm not programmed to do that one.\n");
                                        break;
                                }  
                                
                                return (0);
                            }

Arrays

Arrays are a type of data storage often used in C programs.

An array is a collection of data storage locations, each having the same data type and the same name.

Each storage location in an array is called an array element.

A good example of an array would be storing 12 elements as array elements instead of declaring 12 different variables for each element.

Single-Dimensional Arrays.

A single-dimensional array has only a single subscript.

A subscript is a number in brackets that follows on array's name. This number can identify the number of individual elements in the array.

Example of an array of type float:

float expenses[12];

The array is named expenses, and it contains 12 elements. Each of the 12 elements is the exact equivalent of a single float variable.

All of C's data types can be used for arrays.

Array elements are always numbered starting at 0, so the 12 elements of expenses are numbered 0 through 11.

When you declare an array, the compiler sets aside a block of memory large enough to hold the entire array.

Individual array elements are stored in sequential memory locations.

The location of array declarations in your source code is important. As with nonarray variables, the declaration's location affects how your program can use the array.

An array element can be used in your program anywhere a nonarray variable of the same type can be used.

Individual elements of the array are accessed by using the array name followed by the element subscript enclosed in square brackets.

For example, to store a value in the second array element, we would use:

expenses[1] = 89.95;

We can also assign a copy of the valuue that is stored in an array element. Like:

expenses[10] = expenses[11];

Input / Output

Standard I/O

This is an input/output stream construct for command line programs.

It's common in all Unix and Unix-like systems.

C has no special syntax for input/output. Instead, C defines a library of functions that support integration with standard input/output.

printf()

displays formatted output to stdout.

The printf() function has no return value defined, and we pass a string, wrapped in double quotes.

Somewhere inside the stdio library, printf is defined as: int printf(const char *format, ...);

syntax is: printf(format_str, arg1, arg2, ...);

arguments are the variables to be printed.

format string tells printf() how to display these arguments.

scanf()

reads formatted input from standard input.

syntax is: scanf(format_str, &arg1, &arg2, ...);

arguments are the address of variables that will store the read values.

format string tells scanf() how the input is formatted.

getchar()

used for reading input.

Reads the next character from standard input (if available).

The function returns EOF when end of file or error conditions are encountered.

putchar()

Writes the character c (converted to an unsigned char) to standard output.

Returns the character that was written.

Header Files

Simple programs can be put in a single file. But when a program grows larger, it's impossible to keep it all in just one file.

You can move parts of a program to a separate file, then create a header file.

A header file looks like a normal C file, except it ends with .h instead of .c

Instead of implementations of your functions and other parts of a programs, it holds the declarations.

A good example of a header file is the: #include <stdio.h>

#include - is a preprocessor directive.

The preprocessor goes and looks up the stdio.h file in the standard library because you used brackets around it.

To include your own header files, use quotes:

#include "myfile.h"

This looks up myfile.h in the current folder.

You can also use a folder structure for libraries:

#include "myfolder/myfile.h"

An example to illustrate the use of header files:

Say we have a C program to calculate the years since a given year named main.c:

                        
                            #include <stdio.h>
                            int calculateAge(int year)
                            {
                                const int CURRENT_YEAR = 2020;
                                return CURRENT_YEAR - year;                                
                            }

                            int main(void)
                            {
                                printf("%u", calculateAge(1983));
                            }

Support we want to move the calculateAge function to a separate file. We'd create a calculate_age.c file.

                        
                            int calculateAge(int year){
                                const int CURRENT_YEAR = 2020;
                                return CURRENT_YEAR - year;
                            }

We then need to create calculate_age.h file where we put the function prototype, which is the same as the function in the .c file, except theh body.

                        
                            int calculateAge(int year);

In the main.c file, we can go ahead and remove the calculateAge() function defintion, and import calculate_age.h which will make calculateAge() function available:

                        
                            #include <stdio.h>
                            #include "calculate_age.h"

                            int main(void)
                            {
                                printf("%u", calculateAge(1983));
                            }

To compile a program like this made up of many different files, we need to list them all in the command line like:

gcc -o main main.c calculate_age.c

With more complex setups, a Makefile is necessary to tell the compiler how to compile the program.
