CS concepts for interviews

This requires you to explain the concepts in great detail so you need to have a good understanding.

Most code bootcamps just teach you how to code according to industry standards but not so much of the computer science aspects which are extremely important.

The concepts include:

  • Multithreading. Understand what threads are, arrays conditions and deadlock
  • Pros / Cons of Data Structures. Like why use arrays instead of a list, why a hashmap instead of an array.
  • Garbage Collection. Understand memory allocation.
  • Database Efficiency. Look at clustered indexes, how efficiently are you writing SQL scripts, using schemas in your tables.
  • Idempotency. Where a process even though receiving multiple request only returns one request instead of multiple responses.

Learning a New Language

When learning a new language, look at the:

  1. Learn the basics and fundamentals (flow of a program, expressions, statements, statement blocks, functions, errors)
  2. looks at concepts such as: (Data types[bit, byte, short, integer, long, float, double, boolean, char, primitive vs Non-Primitive], Data structures[arrays, arraylists, linkedlist, stack, queues, graphs, trees, hashtable, sets, heaps])
  3. Learn object-oriented programming.(Classes, abstract classes, interfaces) then core concepts of OOP(Inheritance, encapsulation, polymorphism, abstraction)
  4. learn algorithms(recursion, searching algorithms[depth first search, breath first search, binary search], sorting algorithms[merge sort, quick sort, insertion sort, selection sort])
  5. learn web development(HTML, CSS, Javascript, React, Angular, Vue)
  6. learn git and GitHub and upload your projects
  7. work on simple projects as you're learning.(helps practice the fundamentals and optimization of code)
  8. Syntax of the language
  9. variable types (esp in c but not really all languages)
  10. how to declare variables
  11. how to initialize variables
  12. how to use the variables in the program
  13. how to write expressions
  14. how to write statements

General Tips

Both Web Dev and Software Eng. involve programming and craftsmanship but software engineering adds to it knowledge of data structures and algorithms, compiled languages, memory considerations and understanding the impact of coding and architecture decisions on the machines where they reside.

Large companies that hire software engineers expect candidates to have this knowledge.

Companies like google set a very high bar for hiring. They want to hire only the best. So if you set your sight high (getting hired at big Tech companies) you still be quite hireable elsewhere.

Big companies reward merit, encourage big ideas and give employees the freedom to make good decisions for the user.

The companies hire smart and passionate people base on coding ability amd technical knowledge.

Like any startup with a big goal, you make mistakes and do things that waste time.

It's good to ask, not assume.

Start coding problems sooner.

Space repetition is the key to memorization. Once you learn something, review it again later and again even later. At each repetition you reinforce learning. Spending hours and hours at one time on priority ques won't make you an expert. You become an expert by revisiting and reviewing over time. This get you to a point you can't forget details.

Know algorithms that will save computing resources of time, space and I/O.

Web Development Roadmap

Roadmap

  1. Choose your path:
    • Frontend Dev
      • is the front facing website or app UI
      • learn: HTML (HTML 5 and semantic HTML), CSS (Flexbox, Grid, Animations, Transitions and Responsive Design), Sass, JavaScript (Classes, High order array methods, getting data from backend + JSON for getting data to front-end), TypeScript, JS Frameworks (react), DOM, HTTP, etc.
      • front-end tools include:
        1. Text editor
        2. Browser Development Tools
        3. Version control
      • Other Technologies to learn:
        1. Sass / CSS preprocessor
        2. JS libraries
        3. JS Frameworks (JQuery)
        4. Command line
        5. Module loading / bundling tools (Webpack, browserify, gulp to minify code so it's loaded quickly and efficiently into the browser)
        6. Wordpress
        7. Image editors (photoshop)
        8. Wireframe tools (Plans or Sketches e.g. Figma)
    • Backend Dev
      • Deals with server APIs, data, tasks
      • learn: Backend languages (JS, Python, C#, PHP), Databases (Relational, NoSQL), Knowledge of servers and config
    • Full Stack Dev - is the combined frontend and backend
  2. What is your Goal (or Goals)?
    • Professional dev at a company (large or small)
    • Freelancer / consultant / your own business
    • build your own apps / SaaS (Software as a Service)
    • content creation
    • just a hobby
  3. Getting Started : Basic Tools
    • Computer & OS - MacOS, Windows, Linux
    • Text Editor - PHP Storm, VS Code, VIM, Atom, Sublime Text
    • Web Browser - Chrome, Firefox, Safari, Edge, Brave
    • Terminal - Default, Hyper, Powershell, Git Bash
    • Design Software - Adobe XD, Photoshop, Sketch, Figma, Photoshop.
  4. The Building Blocks: HTML & CSS
    • HTML 5 Page Structure & semantic Tags
    • Basic CSS Styling
    • Positioning
    • Alignment (Flexbox & CSS Grid)
    • Transitions & Animations
    • Responsive Design / Media Queries
    • Sass - CSS preprocessor that gives you more functionality in your styling. Sass offers things like variables, mixins, functions, nesting etc.
  5. CSS / UI Frameworks
    • Tailwind CSS - utility-first framework
    • Bootstrap - popular framework (bootstrap 5 released)
    • Materialize - based on material design
    • Bulma - modular & lightweight
  6. UI Design Principles and Practices (look at Gary Simons video)
    • Color & Contrast - make sure text is readable
    • White Space - spacing between elements
    • scale - sizing relative to other elements
    • visual hierarchy - arrange in order of importance
    • Typography - text typefaces, sizing etc.
  7. JavaScript - extremely important for frontend / fullstack web developers. It's the language of the client side.
    • Basics - variables, arrays, functions, loops etc.
    • DOM & Styling - selecting and manipulating elements
    • Array Methods - foreach, map, filter, reduce, etc
    • JSON - JavaScript Object Notation
    • HTTP Requests - Fetch API - GET, POST, PUT, DELETE
  8. Other Tools to Start to Learn
    • Version Control - Git, Subversion
    • Repo Manager - GitHub,Bitbucket, GitLab
    • Package Manager - NPM, Yarn
    • Module Bundler (FE) - Parcel, WebPack, Rollup
    • Browser Developer Tools (Console, Network, Storage, etc)
    • Editor Extensions & Helpers (linting, Prettier, Live Server, Emmet, Snippets, etc)
  9. Basic Frontend Deployment - be able to do a basic website or frontend app deployment
    • Static Hosting - Netlify, GitHub Pages, Heroku
