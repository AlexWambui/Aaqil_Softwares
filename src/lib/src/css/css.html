Border Style

  • ridge : 3D ridged border. The efffect depends on the border-color value

Margins

used to create space around elements, outside of any defined borders.

There are properties for setting the margin for each side of an element (top, right, bottom, and left).

All the margin properties can have the following values: (Negative values are allowed.)

  • auto : the browser calculates the margin
  • lenght : specifies a margin in px, pt, cm etc.
  • % : specifies margin in % of the width of the containing element
  • inherit : specifies the margin should be inherited from the parent element

margins

Code:

<!-- CSS -->    
p.one_margins, 
p.two_margins,
p.three_margins,
p.four_margins{
    border: 2px solid black;
}
p.one_margins{
    margin: 25px;
}
p.two_margins{
    margin:5%;
}
p.three_margins{
    margin:25px auto 4px;
}
p.four_margins{
    margin-left:5%;
}

<!-- HTML -->
<p class="demo one_margins">margin: 25px;</p>
<p class="demo two_margins">margin: 5%;</p>
<p class="demo three_margins">margin: 25px auto 4px;</p>
<p class="demo four_margins">margin-left: 5%;</p>

Output:

margin: 25px;

margin: 5%;

margin: 25px auto 4px;

margin-left: 5%;

auto value

You can set the margin property to auto to horizontally center the element within its container.

The element will then take up the specified width, and the remaining space will be split equally between the left and right margins.

auto value

Code:

<!-- CSS -->   
.auto_value{
    width: 300px;
    margin:auto;
    border: 1px solid blue;
}
.auto_value p{
    text-align:center;
}

<!-- HTML -->
<div class="auto_value">
    <p>margin: auto;</p>
</div>

Output:

margin: auto;

margin collapse:

Top and bottom margins of elements are sometimes collapsed into a single margin that is equal to the largest of the two margins.

This does not happen on left and right margins! Only top and bottom margins!

margin collapse

Code:

<!-- CSS -->   
p.one_margin_collapse,
p.two_margin_collapse{
    border: 1px solid blue;
}
p.one_margin_collapse{
    margin: 0 0 20px 0;
}
p.two_margin_collapse{
    margin: 20px 0 0 0;
}

<!-- HTML -->
<p class="one_margin_collapse">margin: 0 0 20px 0;</p>
<p class="two_margin_collapse">margin: 20px 0 0 0;</p>

Output:

margin: 0 0 20px 0;

margin: 20px 0 0 0;

In the example above, the <p> element has a bottom margin of 20px and the second <p> element has a top margin set to 20px.

Common sense would seem to suggest that the vertical margin between the <p> and the second <h2> would be a total of 40px (20px + 20px). But due to margin collapse, the actual margin ends up being 20px.

Grid

to use grid we first set the display property to grid to enable us access all other properties of the grid.

grid

.container{
    display:grid;
    /*add columns: 3 columns*/
    grid-template-columns: 100px 100px 100px;
    /*add rows: how the tall the rows will be*/
    grid-template-rows: 50px 50px;
}

instead of using pixels in settting the width of the columns, fr ( which rep the fraction of the available space).

we can also use auto to set the size of the column to the available space

grid using fr

.container{
    display:grid;
    /*1st takes the available space, 2nd takes 50px, 3rd takes 10 percent of the container, 4th and 5th take the fraction of the 3 sections - 4th takes 2 / 3 and 5th takes 1 / 3 of the fraction*/
    grid-template-column: auto 50px 10% 2fr 1fr;
    grid-template-rows:50px 50px;
}

to create a column gap use: grid-column-gap:2px;

to create a row gap use: grid-row-gap: 4px;

grid column / row gap

.container{
    display:grid;
    /*1st takes the available space, 2nd takes 50px, 3rd takes 10 percent of the container, 4th and 5th take the fraction of the 3 sections - 4th takes 2 / 3 and 5th takes 1 / 3 of the fraction*/
    grid-template-column: auto 50px 10% 2fr 1fr;
    grid-template-rows:50px 50px;
    grid-column-gap:5px;
    grid-row-gap:4px;
}

the short hand for grid gap is grid-gap:4px; for both column gap and row gap.

when we use two values; grid-gap: 4px 2px; the first value becomes the row gap and the second the column gap.

grid column / row gap shorthand

.container{
    display:grid;
    /*1st takes the available space, 2nd takes 50px, 3rd takes 10 percent of the container, 4th and 5th take the fraction of the 3 sections - 4th takes 2 / 3 and 5th takes 1 / 3 of the fraction*/
    grid-template-column: auto 50px 10% 2fr 1fr;
    grid-template-rows:50px 50px;
    grid-gap:5px 10px;
 }

Media Queries and Responsive Web Design

Media queries allow to express that certain CSS rules should only be executed when certain conditions are in place (such as based on the screen’s resolution, or the number of colors a graphics card can handle).

But the one condition that started to get very used, is the one allowing to only execute some CSS depending on the browser’s width. As you may have understood, this allows a single CSS codebase for a website regardless of the size of the browser, enabling and disabling some CSS rules when it passes certain width thresholds.

Do you remember Jeffrey Zeldman? (He’s the guy who founded A List Apart, the Web Standards Project, and wrote Designing with Web Standards) He’s part of the people who actually came up with the notion of Responsive Web Design, the idea to use media queries to make a single website adapt depending on the browser’s width.

for CSS, instead of media queries looking for a type of device, they look at the capability of the device.

media queries check:

  • Width and height of the viewport
  • Width and height of the device
  • Orientation (the tablet / phone in landscape or portrait mode?)
  • resolution

Media Query Syntax

A media query consists of a media type and can contain one or more expressions, which resolve to either true or false.

media query syntax

@media not|only mediatype and (expressions) {
  CSS-Code;
}

The result of the query is true if the specified media type matches the type of device the document is being displayed on and all expressions in the media query are true.

When a media query is true, the corresponding style sheet or style rules are applied, following the normal cascading rules.

Unless you use the not or only operators, the media type is optional and the all type will be implied.

You can also have different stylesheets for different media:

linking different style sheets

 <link rel="stylesheet" media="mediatype and|not|only (expressions)" href="print.css">

Media types

Value Description
all used for all media type devices
Print used for printers
Screen Used for computer screens, tablets, smart-phones etc.
speech used for screenreaders that "reads" the page out loud

Simple Examples

One way to use media queries is to have an alternate CSS section right inside your style sheet.

The following example changes the background-color to lightgreen if the viewport is 480 pixels wide or wider (if the viewport is less than 480 pixels, the background-color will be pink):

media queries example

body {
  background-color: pink;
}    

@media screen and (min-width: 480px) {
  body {
    background-color: lightgreen;
  }
}

The following example shows a menu that will float to the left of the page if the viewport is 480 pixels wide or wider (if the viewport is less than 480 pixels, the menu will be on top of the content):

viewport = 480px

@media screen and (min-width: 480px) {
  #leftsidebar {width: 200px; float: left;}
  #main {margin-left: 216px;}
}
