Getting started

Prerequisites

Some of the things you should know in advance before you start React include:

  1. Familiarity with HTML & CSS
  2. Basic knowledge of JS and programming
  3. Basic understanding of the DOM
  4. Familiarity with ES6 syntax and features
  5. Familiarity with Babel (A JS compiler that lets us use ES6+ in old browsers) and Webpack.
  6. Node.js and npm installed globally

To check whether Node.js is installed, use the command:

node -v

What is React?

React, sometimes referred to as a frontend JavaScript framework, is a JavaScript library created by Facebook as an open source project.

React is a tool for building UI components.

React is a declarative, efficient and flexible JavaScript library for building User Interfaces.

React is the view layer of an MVC application (Model View Controller).

React let's you compose complex UIs from small and isolated pieces of code called "components".

Components are like custom, reusable HTML elements, to quickly and efficiently build user interfaces.

React has a few different kinds of components. eg. React.Component subclasses.

We use components to tell React what we want to see on the screen. When our data changes, React efficiently updates and re-renders our components.

A component takes in parameters called props (short for "properties"), and returns a hierachy of views to display via the render method.

The render method returns a description of what you want to see on the screen.

React takes the description and displays the result. In particular, render returns a React element, which is lightweight description of what to render. Most React developers use a special syntax called "JSX" which makes these structures easier to write.

The <div /> syntax is transformed at build time to React.createElement('div').

JSX comes with the full power of JavaScript. You can put any JS expressions withing braces inside JSX.

Each React element is a JS object that you can store in a variable or pass around in your program.

Each React component is encapsulated and can operate independently; This allows you to build complex UIs from simple components.

Setup and Installation

There's a few ways to set up React.

  • Static HTML file - this method is not a popular way to set up React but it's familiar and easy to understand if you've ever used a library like jQuey, and it's the least scary way to get started if you're not familiar with Webpack, Babel and Node.js.
  • Create React App - this is an enviroment that comes pre-configured with everything we need to build a React app.

Create React App

Create React App method of creating React apps creates a live development server, uses Webpack to automatically compile React, JSX and ES6, auto-prefix CSS files, and use ESLint to test and warn about mistakes in the code.

To set up create-react-app, run the following in your terminal, one directory up from where you want the project to live.

npx create-react-app .

After downloading the folders needed for our application, we need to take note of some commands:

  • npm start - used to start the development enviroment. Once we run the command "npm start", a new window will popup at localhost:3000 with a new React app.
  • npm run build - builds our application, minifies all files and gives us a production version that we can deploy to a server and access on the internet.

Now in index.js, we import React, ReactDOM and the CSS file.

We load the Component as a property of React so we don't need to extend React.Component. 

								
									import React, &3123;Component} from 'react'
									import ReactDOM from 'react-dom'
									import './styles/styles.css'

Then we create our App component again.

Notice we use className instead of class for CSS classes, because we're writing JS and class is a reserved word in JS. 

								
									class App extends Component {
										render() {
										  return (
										    <div class="App">
										      <h1>Hello, React!</h1>
										    </div>
										  )
										}
									}

Finally, we render the App to the root.

								
									ReactDOM.render(<App />, document.getElementById('root'))

The full index.js code.

								
									import React, {Component} from 'react'

Elements

We write React elements using a feature called jsx.

React elements are written just like regular HTML elements. You can write any valid HTML element in React.

However, because JSX is really just like JS functions (and not HTML), the syntax is a bit different.

Unlike HTML, single-tag elements (like img element), must be self-closing. ie. must end with a forward slash ( / ).

Fragments

React gives us an element called a fragment.

React requires all returned elements be returned within a single "parent" component.

For example, we can't return two sibling elements, like a h1 and a paragraph from a component.

								
									//this syntax is invalid
									function MyComponent(){
										return(
											<h1>Header</h1>
											<p>Paragraph</p>
										);
									}

the correct code would be:

								
									//valid syntax
									function MyComponent(){
										return(
											<>
												<h1>Header</h1>
												<p>Paragraph</p>
											</>
										);
									}

We can write fragments in a regular or shorthand syntax: <React.Fragment></React.Fragment> or <></>.

Element Attributes

JSX also requires a different syntax for its attributes.

Since JSX is really JS and JS uses a camelcase naming convention, attributes are written differently than in HTML.

The most common example is the class attribute, written as className.

								
									<div class="container"></div>

Element Styles

To apply inline styles, instead of using double quotes(""), we use two sets of curly braces {{}}

Inline styles are not written as plain strings, but as properties on objects.

								
									<h1 style={{fontSize: 24, margin: '0 auto', textAlign: 'center'}}>Header</h1>

Components

React is all about components.

Components refer to small independent parts of your app pertaining to a specific functionality and UI [ not necessarily] that can be re-used throughout your app.

For example: button, header, footer components.

Components can be divided into two categories:

  • Containers / Stateful Components - are the components which have a direct subscription to the state of the app, likely a store. They have access to the values in the state and can trigger changes to the state.
  • Presentational / Stateless Components - are the components who have an indirect access / subscription to the state of the app. They are used only to display information passed through props and cannot trigger any change to the state directly.

a Container contains one or more Presentational Components or, sometimes a Container as its children. However, A Presentational Component can contain only other Presentational Components as its children.

if we need to trigger a state change when we click a button in the Presentational Component? We pass the function that triggers the state changes as a prop from the container and call the function inside the Presentational Component.

A basic function component is written similarly to a regular JS function with a couple of differences such as:

  • Component names must start with a capital letter (ie. MyComponent instead of myComponent).
  • Components, unline JS functions, must return JSX.
                                                                
                                                                        function App(){
										return(
											<div class="App">
												<p>Hello, React!</p>
											</div>
										);
									}

Props

React components can accept data passed to them called props.

Passing props is how information flows in React apps, from parent components to children components.

Example: to pass a prop name from App to the User component.

								
									function App(){
										return <User name="John Doe" />
									}
									function User(props){
										return <h1>Hello, {props.name}</h1>; //Hello, John Doe!
									}

Routing

React-router-Dom helps us navigate to different components by changing the URL to Navigate Page in ReactJS.

Single page applications

These applications can work in different ways. One is by downloading the entire site's contents all at once. This way, when navigating around on the site, everything is already available to the browser, and it doesn't need to refresh the page. Another way is by downloading everything that's needed to render the page the user requested. Then when the user navigates to a new page, asynchronous JS requests are made for just the content that was requested.

Another key factor in a good single-page app is that the URL control the page content. Single-page apps are highly interactive, and users want to be able to get back to a certain state using just the URL.

React Router

turns react projects into single-page applications.

It does this by providing a number of specialized components that mange the creation of links, manage the app's URL, provides transitions when navigating between different URL locations and so much more.

It's an open source package available on the node package manage.

Github repo: https://github.com/ReactTraining/react-router

to install react-router-dom use the command:

npm i react-router-dom

However, some versions of react don't work with features like switch so the best react-router-dom to install would be version 5.3.0 which can be installed using:

npm i react-router-dom@5.3.0

To uninstall react-router-dom, use:

npm uninstall react-router-dom

The major factor of React Router Dom are:

  • BrowserRouter
  • Link
  • Route

BrowserRouter

Works on the change in the URL. As the URL changes it responds and renders the given instruction or pages. But for BrowserRouter to work we need to wrap the whole App component inside the Browser router this itself is done in two ways.

One is to wrap the app component inside the index.js in BrowserRouter like:

								
									<BrowserRouter>
										<App />
									</BrowserRouter>

Or wrap every component in App.js file inside the BrowserRouter like:

								
									function App(){
										return(
											<Router>
												<>
													<Navbar />
													<Route exact path = "/" component = &123;landing} />
												</>
											</Router>
										);
									}

									export default App;

Link Component

In React, the anchor tag does not work to navigate to different pages of the web app. Hence we use the Link component from React Router Dom.

Link is a straightforward way to provide declrative, accessible navigation around your application.

By passing to Link's property to the Link component, you tell your app which path to route to.

Syntax of link:

								
									<Link to="/about">About</Link>

Route Component

Route is the main factor that recognizes the path and makes the routing possible. The route takes the path as attribute / property and loads the component that is passed in the component attribute.

Syntax:

								
									<Route exact path = "/" component=&123;Landing} />

To use props / state to be passed inside the Route, we must use the render function. This function will load the component along with the Property.

Syntax:

								
									<Route exact path="/" render = { () => {
										<Welcome name="Aaqil Softwares" />
									}} />

Tips

Using bootstrap

First install bootstrap using the command

npm install bootstrap

Then import bootstrap in the index.js file or App.js file using

import “bootstrap/dist/css/bootstrap.min.css”;

Using Sass

To use Sass in React, first install node sass using the command:

npm install node-sass

Once node-sass is installed, use .scss files instead of .css files in your imports. e.g import './App.scss' instead of './App.css'
