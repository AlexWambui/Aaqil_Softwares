Introduction

What is it?

Software Engineering is not the same as web development.

Software Engineering is the process of designing and building computer programs [operating systems, mobile apps and websites] and systems. It involves writing code, testing and debugging software, and maintaining and updating it over time.

In simple terms, Software engineering is all about using computers and code to build useful and fun things people can use.

A good example is when you want to build a computer game. You need to come up with the idea for the game, plan out how it will work, use a programming language to write the code that will make the game run. Once the game is finished, you would need to test it to make sure it's working correctly. If you find any problems, you would need to go back and fix them by updating the code.

Large software companies like Google, Amazon, Facebook and Microsoft view software engineering as different from web development. For example, Amazon has Frontend Engineers (FEE) and Software Development Engineers (SDE).

Large companies require computer science knowledge for software development / engineering roles.

some of the things you should know are: stack and heap, Big-O notations, trees, how to traverse a graph, coding a sorting algorith, building your own data structures, managing memory, multidimensional arrays, associative arrays.

Requirements

  1. A little experience with coding (variables, loops, methods / functions, etc)
  2. Good problem solving skills.
  3. Strong analytical and technical skills.
  4. Patience
  5. Time

How to become a software Engineer

  1. Learn computer science and software engineering topics such as algorithms, data structures, computer organizations and programming languages. It's advisable to get certified as some organization would require you to be.
  2. Gain practical experience through internships, co-op programs or by participating in hackathons and coding challenges.
  3. Develop your skills by taking online courses, reading technical blogs and articles and participating in professional development opportunities.

Community

Why is community so important:

  1. The network - helps you get to know people, engage with them and learn with them and from them.
  2. The empowered journey - community empowers you to lead your learning journey. With the internet and the availability of learning assets, you no longer need a "sage on stage" to learn new skills and concepts. Own your learning.
  3. Accountability - a community holds each other accountable. Solo online learning can be intimidating and hard to sustain without a proper support system. With a community, you get to share assets, regularly support each other, reach out for help, and provide feedback to one another.
