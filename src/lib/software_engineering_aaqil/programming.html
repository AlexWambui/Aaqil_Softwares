Learning a New Language

When learning a new language, look at the:

  1. Learn the basics and fundamentals (flow of a program, expressions, statements, statement blocks, functions, errors)
  2. looks at concepts such as: (Data types[bit, byte, short, integer, long, float, double, boolean, char, primitive vs Non-Primitive], Data structures[arrays, arraylists, linkedlist, stack, queues, graphs, trees, hashtable, sets, heaps])
  3. Learn object-oriented programming.(Classes, abstract classes, interfaces) then core concepts of OOP(Inheritance, encapsulation, polymorphism, abstraction)
  4. learn algorithms(recursion, searching algorithms[depth first search, breath first search, binary search], sorting algorithms[merge sort, quick sort, insertion sort, selection sort])
  5. learn web development(HTML, CSS, Javascript, React, Angular, Vue)
  6. learn git and GitHub and upload your projects
  7. work on simple projects as you're learning.(helps practice the fundamentals and optimization of code)
  8. Syntax of the language
  9. variable types (esp in c but not really all languages)
  10. how to declare variables
  11. how to initialize variables
  12. how to use the variables in the program
  13. how to write expressions
  14. how to write statements

Programming Cycle

Preparing to program

There's usually certain steps to take when solving a problem.

First, define the problem. If you don't know what the problem is, you can't find a solution!

Devise a plan to fix it - Once you know what the problem is, you can devise a plan to fix it.

Implement the plan - Once you have a plan, you can usually implement it.

Test the results - Once the plan is implemented, you must test the results to see whether the problem is solved.

The sequence of steps is:

  1. Determine the objective(s) of the program.
  2. Determine the methods you want to use in writing the program - this includes knowing what information needs to be tracked, what formulas will be used, what you need to know and in what order the solution should be implemented.
  3. Create the program to solve the problem.
  4. Run the program to see the results.

The Program Development Cycle

The program development cycle has its own steps.

  1. use an editor to create a disk file containing your source code.
  2. compile the source code to create an object file.
  3. link the compiled code to create an executable file.
  4. run the program to see whether it works as originally planned.

Programmer

programmers tell computers what to do. Computers require these instructions to be precise and complete in every way.

Programmers think in an unnatural way because they break down tasks into small steps and that's not the way the brain works.

Humans aren't usually good at giving precise and complete instructions since we have this incredible brains which lets us give vague commands and still get correct answers. Thus programmers have to think in an unnatural way.

They have to learn to take a description of a task and break it down into the fundamental steps which a computer needs to know in order to perform the task.

Traits of a good Programmer

  • Logical - being logical helps organise and construct complex programs and to break a task down into necessary individual steps.
  • Patient
  • Perceptive - this is to notice that the 20 consecutive hours you've spent in front of your computer have depleted your body's energy resources to nearly fatal levels and you'll die whilst designing an icon for your new application. (Avoid this problem by designing the icon before writing the program).
  • At least moderately intelligent - and also be a star trek fan.
  • Enjoys an intellectual challenge - this helps you to enjoy programming as it can be intellectually challenging.
