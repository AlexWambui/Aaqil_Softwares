Introduction

Laravel is a web application framework with expressive, elegant syntax and was created in 2011 by Taylor Otwell.

It uses the MVC model.

A web framework provides a structure and starting point for creating your application, allowing you to focus on creating something amazing while the framework sweats the details.

Laravel provides powerful features such as:

  • thorough dependency injection.
  • an expressive database abstraction layer.
  • Queues and scheduled jobs.
  • Unit and integration testing.

Installing Xampp

To install Xampp:

  1. download Xampp from apachefriends.org
  2. Once downloaded, navigate to the Downloads folder and run: chmod +x xampp-linux-x64-8.1.12-0-installer.run to make the file executable.
  3. Run the installer using the command: sudo ./xampp-linux-x64-8.1.12-0-installer.run then run through the steps for the installer.
  4. Xampp gets installed into /opt/lampp. Uncheck the launch Xampp box from the installer window then run: cd /opt/lampp
  5. Open Xampp's dashboard using: sudo ./manager-linux-64.run then navigate to the second tab - Manage Servers.

First Project

Before creating your first laravel project, ensure you have PHP and composer installed.

To create a new project (say first_project) via the composer create-project command:

composer create-project laravel/laravel first_project

You can also install the laravel installer globally so as to use the command laravel new project_name command.

To install laravel installer globally, use the command:

composer global require laravel/installer

To create a new laravel project using the laravel installer, use the command:

laravel new project_name

After the project has been created, cd into the project and start Laravel's local development server using the Laravel's Artisan CLI serve command:

php artisan serve

After starting the application, it usually available on the browser using url: http://localhost:8000

Migrations

In laravel, migrations are used to define and manage changes to the database schema over time.

Migrations are stored in the 'database/migrations' directory and contain PHP code that is used to create and modify database tables and columns.

The command used for migrations is:

php artisan migrate

Once the command is run, Laravel will look at the migration files in the database/migrations directory and apply any unapplied migrations to the database. It creates tables or columns that don't yet exist in the database, and also updates the database schema to match the current migration files.

Cheat Sheet

php artisan commands:

  1. php artisan about - quick overview of your application's configuration, drivers and environment.
  2. php artisan about --only=environment - to filter for a particular section of the application overview.
  3. php artisan migrate - used to apply database migrations to the application's database.

Error resolves:

When trying to migrate a database and run into Class "PDO" not found, first run the command php -m to check if the PDO module is present in the installed PHP modules.

If not present, run this command to install php-pdo module (use `apt` instead of `dnf` for debian based distros):

sudo dnf install php-pdo

Now try to run the migrate command again:

php artisan migrate
